# チラシ配布実績管理アプリ - 要件メモ

元資料: `チラシ配布実績管理アプリ_要件メモ.docx` をリポジトリ内に整理したもの。

## 1. 概要

大島てる的に、地図上のピンでチラシ配布実績を記録・管理するアプリ。スタックは「きみのアーカイブ」と同様（Next.js + Supabase + Vercel）。
配布場所（同一地点への複数回訪問を想定）と配布履歴を分離したデータ設計とし、履歴ごとの評価によってピンの色を出し分ける。

## 2. デプロイ構成

| 項目 | 内容 |
|---|---|
| ホスティング | Vercel |
| DB / 認証 | Supabase |
| ドメイン | `flyer.<取得済みドメイン>` |
| DNS設定 | お名前.com Navi でサブドメイン `flyer` の CNAME を Vercel に向ける |
| 既存LP | レンタルサーバーはそのまま。本アプリとは無関係 |

## 3. 認証・編集権限

- Supabase Auth でログイン制（**マジックリンク**を採用）
- Row Level Security (RLS) で未ログインユーザーは一切データ取得不可
- 未ログイン時はログイン画面へリダイレクト
- `visits` に `user_id` を持たせ「誰がいつ配布したか」を記録
- 編集・削除は入力者本人以外のメンバーでも自由に行える（チームで共有・修正し合う運用）
- ただし編集・削除時は必ず `visit_logs` に操作記録を残す（誰が・いつ・何を・どう変えたか）
- 削除は物理削除ではなく**論理削除**（`is_deleted` フラグ）で、誤操作から復元可能
- 一覧に「変更履歴を見る」導線を用意し、各履歴カードから変更ログを遡れる

## 4. データベース設計

### 4-1. locations（配布場所マスタ）
同一地点（学校＋立ち位置）への複数回の配布履歴をまとめる単位。

| カラム | 型 | 説明 |
|---|---|---|
| id | uuid | 主キー |
| school_id | uuid | schools への外部キー |
| spot | text | 立ち位置（例: 西側、正門西側、丁字路右側） |
| label | text | 任意の場所ラベル（建物名など） |
| lat / lng | float | 地図座標（緯度経度） |
| created_at | timestamp | 作成日時 |

### 4-2. visits（配布履歴）
1つの location に対して複数回登録される個別の配布実績記録。

| カラム | 型 | 説明 |
|---|---|---|
| id | uuid | 主キー |
| location_id | uuid | locations への外部キー |
| user_id | uuid | 配布した人（Supabase Auth） |
| date | date | 配布日 |
| start_time | time | 配布開始時刻 |
| end_time | time | 配布終了時刻 |
| count | integer | 配布枚数 |
| rating | text | 評価: good / normal / bad（良好・普通・非推奨） |
| memo | text | メモ・所感 |
| is_deleted | boolean | 論理削除フラグ（物理削除は行わない） |
| created_at | timestamp | 登録日時 |
| updated_at | timestamp | 最終更新日時 |

### 4-3. visit_logs（配布履歴の変更ログ）
`visits` の編集・削除操作を全て記録する監査ログ。DBトリガーで自動記録する。

| カラム | 型 | 説明 |
|---|---|---|
| id | uuid | 主キー |
| visit_id | uuid | 対象の visits.id |
| user_id | uuid | 操作を行ったユーザーID |
| action | text | create / update / delete |
| before_data | jsonb | 変更前（create時はnull） |
| after_data | jsonb | 変更後（delete時はnull） |
| created_at | timestamp | 操作日時 |

### 4-4. schools（対象小学校マスタ）
プルダウン選択用の事前登録リスト。表記ゆれを防ぐ。

| カラム | 型 | 説明 |
|---|---|---|
| id | uuid | 主キー |
| name | text | 小学校名（例: 板橋第一小） |
| created_at | timestamp | 作成日時 |

## 5. ピンの色ロジック

各 location のピンの色は、その location に紐づく visits のうち「日付・開始時刻が最も新しい」1件の rating で決定する（直近の評価を最新の正とする）。

| rating | 表示ラベル | ピンの色 |
|---|---|---|
| good | 良好 | 緑 |
| normal | 普通 | グレー |
| bad | 非推奨 | 赤 |

## 6. 画面・機能一覧

### 6-1. 一覧タブ
- 配布履歴を日時降順で一覧表示。各行はインライン編集可能
- 項目: 日付、開始〜終了時刻、対象小学校、立ち位置、配布枚数、評価、メモ
- フィルタ: 期間（開始日〜終了日）／対象小学校（両方同時適用可）
- サマリー: 表示件数、配布枚数合計、地図未紐付け件数
- 「地図に配置」ボタンから地図タブへ遷移し座標を紐付け
- 各履歴カードに「変更履歴」リンク（誰が・いつ・何を変更したか）
- 削除済みは通常非表示。「削除済みを表示」で復元可能

### 6-2. 地図タブ
- 実地図（**Leaflet + OpenStreetMap** を採用）上にピン表示
- ピンクリックでその location の履歴一覧・詳細を表示
- ピンの上に対象小学校名をラベル表示（密集時の見せ方は今後検討）
- 地図クリックで新規 location 登録、または既存 location への履歴追加

### 6-3. 小学校マスタ管理
- 小学校名の追加・削除（プルダウンに反映）

## 7. 試作版からの変更点

- 地図表示は画像アップロード方式 → 実地図API方式へ
- locations と visits を分離したリレーショナル設計へ
- ピンの色は直近の評価を採用
- 評価は3段階（良好／普通／非推奨）
- 小学校は事前登録リストからのプルダウン選択
- 配布履歴に開始・終了時刻を追加
- ログイン認証（Supabase Auth）を追加。未認証時はデータ非公開
- 編集・削除は誰でも可、ただし visit_logs で変更履歴を記録・論理削除方式を追加

## 8. 決定事項・今後の検討

- 地図API: **Leaflet + OpenStreetMap（無料）** を採用
- ログイン: **マジックリンク** を採用
- 変更ログ: **DBトリガーで自動記録** を採用
- ピンラベルの密集時表示: MVPは常時表示。ズーム連動等の対策は今後
- チームメンバーのアカウント発行: 管理者が Supabase ダッシュボードから招待
