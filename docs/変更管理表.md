## 変更管理の運用ルール
- 変更は必ず Git コミット単位で記録し、コミットメッセージに「種別: 概要」を記載する。
- DB スキーマ変更は `supabase/schema.sql` に反映し、冪等（再実行可能）を維持する。
- 破壊的変更（列削除・データ移行等）を行う場合は、本表に「影響範囲」と「移行手順」を追記する。
