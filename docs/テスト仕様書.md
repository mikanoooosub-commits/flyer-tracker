# テスト仕様書（チラシ配布実績管理アプリ）

対象バージョン: master 最新 / 環境: ローカル（localhost:3000）・本番（Vercel）
凡例: 結果欄は OK / NG / 保留 を記入。

## 1. 認証・認可

| No. | 分類 | 前提条件 | 手順 | 期待結果 | 結果 |
|-----|------|----------|------|----------|------|
| A-01 | 認証 | 未ログイン | 任意のURL（/, /list, /schools）にアクセス | /login へリダイレクトされる | |
| A-02 | 認証 | パスワード設定済 | メール＋パスワードでログイン | ログイン成功し地図（ホーム）へ遷移 | |
| A-03 | 認証 | 未登録メール | メールリンク（初回用）で未登録メールを送信 | 「登録されていません」等の案内が出てリンクは発行されない | |
| A-04 | 認可 | 未認証(anon) | REST API を anon キーで直叩き | 401 permission denied（データ取得不可） | |
| A-05 | 認証 | ログイン済 | ヘッダのログアウト | /login へ遷移し、以降アクセス不可に戻る | |
| A-06 | 認証 | 初回（パスワード未設定） | 「初回の方はこちら」→メールリンク送信→**同一ブラウザ**でリンクを開く | ログイン成功し地図（ホーム）へ遷移 | |
| A-07 | 認証 | メールリンクでログイン済 | ヘッダの🔑→新パスワード設定→ログアウト→パスワードでログイン | 設定したパスワードでログインできる | |
