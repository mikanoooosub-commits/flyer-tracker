## セキュリティ確認（実施済み）
- SQL インジェクション: 入力値 `' and 1=1` を投入 → PostgREST のパラメータ化により文字列として扱われ、注入不成立を確認。
- 未認証アクセス: anon キーで REST API 直叩き → 401 permission denied（RLS 有効）を確認。
- 認可: 未ログイン時に全ページが /login へリダイレクトされることを確認。

## 重要度の定義
- 高: 機能停止・セキュリティに関わる。即時対応。
- 中: 一部機能に支障、または開発効率に影響。計画的に対応。
- 低: 軽微・UX 改善レベル。
